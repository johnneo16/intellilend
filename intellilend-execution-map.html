03
Architecture Decision Records
7 key decisions · all accepted
ADR-001
Monorepo with Turborepo + pnpm
Single repo for web + api + packages. Turborepo task caching for fast CI. Shared TypeScript types without separate package publishing.
Accepted
ADR-002
Vercel + Supabase over AWS
Zero infra management for solo developer. Supabase gives PostgreSQL + S3 storage + auth in one. Can migrate to AWS at any time without code changes.
Accepted
ADR-003
Modular Monolith over Microservices
NestJS modules provide identical logical separation to microservices without network overhead, distributed transactions, or separate CI pipelines. Solo dev cannot maintain 5+ repos.
Accepted
ADR-004
Zod Validation for All LLM Outputs
LLMs hallucinate field names/types even with structured output mode. Zod provides runtime safety net. Validation failures trigger retry with clarifying prompt (max 2 retries).
Accepted
ADR-005
Deterministic Rules + LLM Hybrid Underwriting
FOIR, base risk score, and decision bands computed deterministically first. LLM used only for 5C narrative, reasoning, and confidence scoring. Ensures auditability and regulatory compliance.
Accepted
ADR-006
Context-Aware Adaptive Branding System
Each lifecycle stage has its own colour theme via React ThemeContext + Tailwind CSS variables. Same component library, same layout system. Differentiates IntelliLend as product-grade.
Accepted
ADR-007
Custom JWT Auth over Supabase Auth
Guest-to-user conversion flow requires custom state machine. Full control over token payload (include applicationId, userType). Can add Supabase Auth as optional SSO layer later.
Accepted
04
Delivery Risk Register
Identified risks with mitigation strategies
Risk Impact Likelihood Mitigation
LLM outputs are inconsistent / hallucinated fields High Medium Strict Zod schemas on all outputs. Retry logic (max 2). System prompt with explicit JSON format. Fallback to deterministic score if all retries fail.
PDF parsing by LLM is unreliable for non-standard statements Medium High Provide mock JSON bank statement format as alternative input. Document Agent falls back to user-declared income if extraction fails.
Supabase free tier connection limits under load Low Medium Use PgBouncer (Transaction mode). Portfolio project load is minimal. Upgrade to Pro tier ($25/mo) if needed.
Context drift — solo dev loses momentum mid-build High Medium Phase quality gates prevent unfinished code accumulation. Notion sprint board tracks daily progress. Commit daily even if just chore/progress commits.
OpenAI API cost spikes during What-If Simulator usage Medium Low Debounce What-If requests (500ms). Cache responses for identical inputs. Switch to cheaper model (gpt-4o-mini) for What-If simulation.
NestJS on Render cold starts add latency to first underwriting call Low High Add loading state in UI ("AI agents are analysing your documents…"). Render free tier has cold starts — upgrade to $7/mo starter plan for always-on.
