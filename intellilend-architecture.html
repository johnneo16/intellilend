AI Agent Architecture

📄
Document Intelligence Agent
GPT-4o / Claude 3.5 Sonnet
Input
  • Bank statement PDF
  • Supabase signed URL
Output (Zod)
  • monthly_income
  • avg_balance
  • emi_obligations
  • risk_flags[]
  • stability_score
⚖️
Credit Decision Agent
GPT-4o / Claude 3.5 Sonnet
Input
  • User application
  • FinancialData
  • FOIR (computed)
Output (Zod)
  • decision (A/R/Review)
  • risk_score 0–100
  • 5C factors
  • reasoning text
  • eligible_amount
⚙️ FOIR + risk bands computed deterministically first
🎯
Collections Strategy Agent
GPT-4o / Claude 3.5 Sonnet
Input
  • Days Past Due (DPD)
  • DPD bucket
  • Loan status
Output (Zod)
  • action type
  • message text
  • tone: soft/mod/strict

End-to-End Data Flow — Application to Decision

1
Form Submit
User fills 3-field form + PDF upload. POST /applications
API
2
PDF Storage
Bank statement uploaded to Supabase Storage. Signed URL stored in DB.
Storage
3
Document Agent
LLM extracts income, balance, EMIs, risk flags from PDF.
AI Agent
4
FOIR Engine
Deterministic FOIR + base risk score computed. Saved to FinancialData.
DB
5
Credit Agent
LLM produces 5C scores, decision, reasoning, eligible amount.
AI Agent
6
Persist Decision
RiskAssessment saved. Application status updated to APPROVED/REJECTED/REVIEW.
DB
7
Dashboard Render
GET /applications/:id/decision → Intelligence Dashboard with full 5C viz.
Client
